# Contributing to ORIVION

Thank you for your interest in contributing to ORIVION. We welcome contributions from the community to help build the world's first open, modular enterprise intelligence platform.

## Code of Conduct
Before participating, please review our Code of Conduct (to be added). We expect all contributors to follow these guidelines.

## Reporting issues
- Search for existing issues before submitting a new one.
- When filing an issue, include a clear title and description.
- Provide steps to reproduce bugs and your environment details (OS, browser, etc.).

## Pull requests
- Fork the repository and create your branch from `main`.
- Write clear, descriptive commit messages.
- Maintain coding standards: use linting and include tests where possible.
- Update documentation as needed.
- Reference related issues in your pull request description.

## Documentation
- Contributions to the Build Book (`docs` directory) are welcome.
- Follow the Markdown conventions already in place.
- Include citations and diagrams where appropriate.

## Licensing
By contributing, you agree that your contributions will be licensed under the project’s open-source license.
